Hello,
I am writing with a question about the ability to set TxPDO assign options.

I am working with the Beckhoff EL5101 encoder card.  The default TxPDO
assign settings allow only the status, value, and latch parameters through
(0x1A00 inputs).  I would like to change the TxPDO assign so that the
frequency, window, and period parameters are also passed (0x1A02 inputs).

I was attempting to do this with the "ec_SDOwrite()" function as shown
below:

                // Change number of inputs
                TxPDO_000=2;
                int os=sizeof(TxPDO_000);

                ec_SDOwrite(8,0x1C13,00,FALSE,os,&TxPDO_000,EC_TIMEOUTRXM);

                // Verify change
                TxPDO_000=0;
                os=sizeof(TxPDO_000);

ec_SDOread(8,0x1C13,00,FALSE,&os,&TxPDO_000,EC_TIMEOUTRXM);

                // Add input assignment
                TxPDO_002=0x1A02;
                os=sizeof(TxPDO_002);

                ec_SDOwrite(8,0x1C13,02,FALSE,os,&TxPDO_002,EC_TIMEOUTRXM);

                // Verify change
                TxPDO_002=0;
                os=sizeof(TxPDO_002);
                ec_SDOread(8,0x1C13,02,FALSE,&os,&TxPDO_002,EC_TIMEOUTRXM);

The read statements work (I was able to verify the default settings for the
other subindices were as expected), but the write statements did not.  Also,
I have verfied that the ec_SDOwrite() function works for other parameter
changes (able to change frequency window of this card), but not this.  Any
help on this is greatly appreciated.

Sincerely,
RyanN

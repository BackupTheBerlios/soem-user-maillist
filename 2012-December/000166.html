> After removing the CA flag I'm now able to put the device into Operational Mode, I get a process image of appropriate size and I can read SDO objects. However, I have not found a way to read the PDOs.
> ec_send_processdata() returns with a non-zero wkc, however ec_receive_precessdata(...) always returns with zero.
Zero WKC on ec_receive_data() indicates there is no data transfer 
between the master and slave logical IO mapping. Can you send a (zipped) 
wireshark trace of a your device in OP? Also a dump of "./slaveinfo eth0 
-map" would help.
>
> Maybe TxPDO and RxPDO have to be used instead of ec_receive_processdata for CoE? Unfortunately I did not find much information about using CoE with SOEM. Has anyone managed to use synchronous PDOs with SOEM so far?
SOEM is used in hundreds of projects on many master targets and slave 
combinations. It does work as advertised. But sometimes there can be a 
snag before all works out. I always suggest to start simple, for example 
a digital input / output slave. And make a "blinky" project. When that 
works you can go to more complex projects.

hello

i bought a beckhoff EK1100 and a beckhoff EL3192 to test ethercat on my
RT-PREEMPT linux system. first steps involved the IgH EtherCAT Master. i
had trouble getting kernel module stuff to work and am now at a point
where it usually works. if not i have to start and stop the module
twice. then i can analyze a analog input value correctly:

./ethercat upload -p 1 --type int16 0x3101 02

then i wrote my own application to try to get this value in cyclic mode.
i just don't know how to put the device into "OP" mode. there is no call
to do that.

frustrated i tried TwinCAT on windows and also didn't get it doing
something useful - i just wanted to check if my hardware can go to op mode.

googling for "OP linux ethercat" or so, i found SOAM. i downloaded,
fixed the makefile by removing -lpq which is the postgresql client
library which is obviously not needed :-) and compiled, started and
instantly my bus worked, i got analog input values display on screen
without any worries.

that's really great! thanks for that software. i will now include it
into my realtime framework as i currently use a analog input proprietary
pci card, and some usb to canbus converter and still i need more io, i
am now trying ethercat and will try to use soam to attach to it. let's
see if it worx.



thanks!

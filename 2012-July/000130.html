> It is very difficult to find an environment for the master (hardware and OS)
> that has a good system clock. Some embedded systems only have a 32KHz clock,
> some others are not monotonous. SOEM is OS and hardware agnostic so it opts
> for the master-syncs-with-reference-slave option. This is not better or
> worse than the other way around. The fact that EtherCAT documentation seems
> to prefer one method says nothing about the quality of other solutions.

That makes sense -- thanks for the explanation.

> But to implement the reference-slave-syncs-with-master is not difficult.
>
> In pseudo code:
>
> main loop
> {
>     send process data
>     receive process data
>
>     new slave reference time = master time - ethercat offset
>     FPRW(reference slave, 0x910 , new slave reference time)
>
>     do control stuff
>
>     wait for cycle end
> }

That seems completely reasonable. Thanks.

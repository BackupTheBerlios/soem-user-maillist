Hi all,

Has anyone here ever implemented etherCAT across a TCP/IP network? Would
this be supported by SOEM?

According to this article [1]: "If an EtherCAT packet needs to be routed
over a standard TCP/IP network, it can be transported inside a UDP packet,
where it is marked for easy identification using UDP port number 34980
(0x88A4), a number assigned by the Internet Assigned Numbers Authority
(IANA) for transport of EtherCAT packets over TCP/IP networks."

My use case is an installation where we don't have control over the
physical network, and the etherCAT chain must share a part of the network
with a regular LAN.

PC -------------|
SOEM -------|   |
         TCP/IP Switch ------------ TCP/IP Switch
                                     | |     |
                                     | |     ------- PC
                                     | ----- Ethercat Slave A
                                     ------- Ethercat Slave B

Regards,
Ian

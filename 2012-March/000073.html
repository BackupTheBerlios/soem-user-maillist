Lots of slaves do not support the Object Dictionary read functions. To 
see what is supported if it is not clearly stated in the manual read the 
ESI file that belongs to the slave. This XML file is (or should be) an 
accurate description of the slave capabillities. If you have trouble 
parsing it PM the file to me.

Not supporting OD is somewhat of a leazy escape from the vendor. It is 
not mandatory but recommended in the EtherCAT specifications. So try 
complaining at Yaskawa.

Hi everyone!

I'm building a configuration tool that reads the ENI XML file and configures the network.

now I'm trying to configure the process image and I have some problem with the logical addressing.

in the XML file generated by TwinCAT the logical addresses start from 0x01000000 for Inputs and from 0x09000000 for Outputs;

instead, in the SOEM, the logical addresses start from 0x00000000 for group0, 0x00010000 for group1, 0x00020000 for group2 and so on (ethercatconfig.c, function ec_config_init, lines 156 to 159) .

in function ec_config_map_group, this start logical address is updated, assigned to ec_slave[slave].FMMU[FMMUc].LogStart, written to the slave and finally the pointer to the process image for this slave is set by ec_slave[slave].outputs = pIOmap + etohl(ec_slave[slave].FMMU[FMMUc].LogStart (line 683)

here is the problem: if I try to configure FMMUs from the XML file I can't simply assign, for example, ec_slave[slave].outputs = pIOmap +0x09000000, because I will be out of range (consider tha pIOmap is 4096 byte long).

of course I can ignore the values set by TwinCAT, but this lead me to ignore all the init commands regarding the fmmu configuration and also the cyclic frame generated by TwinCAT; basically I have to ignore the XML file!

so, my questions are:

-Why TwinCAT chooses these start logical addresses?
-Could these values be changed?
-Could be found a relation between these value and the ones set in SOEM?

hope my questions are clear!

Thanks a lot,

Rocco

Hi,

I have solved the "problem" with the redundancy test example freezing the
comp. The problem was with the bad kernel configuration of the RTAI Linux
extension. But, when I installed fresh Linux distribution and compiled the
code, app ran perfectly but master could not make the slaves to go into OP
state.

Anyway, for the interested parties, after the inspection of the other
example codes (mainly simple_test.c), I have found the potential bug in the
red_test.c. It is needed to put the following after the line 85 (after
ec_slave[0].state = EC_STATE_OPERATIONAL;):

ec_send_processdata();

The code now runs perfectly and can be tested for various scenarios.
